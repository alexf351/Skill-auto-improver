KEY FEATURES
================================================================================

✅ CHECKLIST DEFINITION
   - Simple JSON format
   - 3-6 yes/no questions
   - Each question: id, question, description, required flag
   - Load from file or dict

✅ EVALUATION MODES
   1. Fixture-only (existing)
   2. Checklist-only (new)
   3. Hybrid either/or (new) - fixture OR checklist passes
   4. Hybrid both required (new) - fixture AND checklist pass

✅ SCORING
   - Per-output: 0-100% (questions_passed / total_questions * 100)
   - Pass rate: total_passed / total_outputs (0.0-1.0)
   - Average score: mean of all output scores (0-100%)

✅ RULE-BASED PATTERNS (No custom code needed)
   - Direct: "q1": true/false in output
   - Field exist: "has_field_NAME" checks for field
   - Non-empty: "is_non_empty_NAME" checks field exists and non-empty

✅ CUSTOM EVALUATOR (For LLM-based evaluation)
   - Pass function(output, question) -> bool
   - Full flexibility for complex logic
   - Exception handling built-in

✅ INTEGRATION WITH PATCH TRIALS
   - Same keep/revert logic as fixtures
   - Before/after comparison for regression detection
   - Works with operating_memory promotion profiles

✅ CLI SUPPORT
   - evaluate-checklist: Run checklist evaluation
   - evaluate-hybrid: Run fixture + checklist
   - --require-both: AND logic for hybrid
   - JSON output for easy parsing

================================================================================
USAGE EXAMPLES
================================================================================

1. RULE-BASED CHECKLIST (No custom code)

   Define checklist.json:
   {
     "name": "quality",
     "questions": [
       {"id": "q1", "question": "Has output?"},
       {"id": "q2", "question": "No errors?"}
     ]
   }

   Define outputs.json:
   {
     "run_1": {"q1": true, "q2": true},
     "run_2": {"q1": true, "q2": false}
   }

   Run:
   python3 -m skill_auto_improver.cli evaluate-checklist \
     --checklist checklist.json \
     --outputs outputs.json

   Result:
   - run_1: 100% (2/2 yes)
   - run_2: 50% (1/2 yes)
   - Pass rate: 50% (1/2 passed all)

2. CUSTOM EVALUATOR (LLM-based)

   from skill_auto_improver.checklist_evaluator import ChecklistEvaluator

   def evaluate_quality(output, question):
       if question.id == "is_concise":
           return len(output.get("text", "")) < 100
       return False

   evaluator = ChecklistEvaluator(checklist, evaluator_fn=evaluate_quality)
   report = evaluator.evaluate_all(outputs)

3. HYBRID MODE

   python3 -m skill_auto_improver.cli evaluate-hybrid \
     --fixtures fixtures.json \
     --checklist checklist.json \
     --outputs outputs.json \
     --require-both  # Both gates must pass
